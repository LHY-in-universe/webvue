================================================================================
                    Windows本地推送步骤（正确版本）
================================================================================

🔍 您遇到的问题：
  - 本地webvue目录是旧版本
  - 没有bench分支
  - 补丁无法应用到旧代码

✅ 解决方案：使用Git直接从服务器拉取

================================================================================
                    正确的操作步骤
================================================================================

第1步: 清理旧目录（在Windows PowerShell或CMD中）
------------------------------------------------
cd C:\Users\22518
rmdir /s /q webvue


第2步: 重新克隆仓库
-------------------
git clone https://github.com/LHY-in-universe/webvue.git
cd webvue


第3步: 切换到bench分支
----------------------
git checkout bench


第4步: 添加服务器作为远程仓库
-----------------------------
git remote add server root@175.178.24.56:/home/webvue


第5步: 从服务器拉取最新代码
---------------------------
git fetch server bench


第6步: 合并服务器的更改
-----------------------
git merge server/bench --allow-unrelated-histories


第7步: 强制推送到GitHub
------------------------
git push origin bench -f


完成！✅
